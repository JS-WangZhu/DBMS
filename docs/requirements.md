# DBMS数据库管理平台需求规格（落地版）

## 1. 项目目标

构建统一数据库管理平台，覆盖 MySQL、Redis、Doris、MongoDB 的集中管理，降低 DBA 运维成本，提升可视化与自动化能力。

## 2. 角色与边界

- 管理员：实例录入、备份配置、用户管理、策略调整。
- 普通用户：仅查看授权实例状态，不允许修改配置或执行备份。
- 系统边界：平台负责管理、监控、调度和审计；数据库恢复操作默认由 DBA 在线下执行（平台提供指引）。

## 3. 功能需求

### 3.1 数据库管理

- 支持 MySQL / Redis / Doris / MongoDB 的独立管理视图。
- 支持单实例与集群录入。
- 录入地址支持 IP 或域名。

### 3.2 MySQL

- 主从拓扑展示：主从关系、IO/SQL线程状态、复制延迟。
- 连接状态：当前连接、最大连接、活跃连接。
- 实时会话：展示运行中的 SQL 线程并支持 kill。
- 运行指标：Uptime、版本、QPS/TPS。
- 只读状态：`read_only`。
- 批量操作：集群内实例配置变更与巡检。

### 3.3 MongoDB

- 支持 mongod/mongos 单实例录入。
- 支持副本集录入：成员角色、优先级、仲裁节点。
- 可选支持分片集群录入。
- 状态监控：角色、健康、optimeDate、复制延迟、连接数、currentOp。

### 3.4 Redis

- 支持单节点、主从、Cluster 模式。
- Cluster 自动拓扑发现（`cluster nodes`）。
- 监控：节点角色、连接数、内存、命中率、key 数量、复制偏移、槽位健康。

### 3.5 域名解析刷新

- 对域名地址进行周期解析（建议 1 小时）。
- 发现解析 IP 变化时更新元数据并记录审计事件。
- 提供手动触发刷新接口。

### 3.6 备份管理（MySQL + MongoDB）

- 备份策略：全量、增量（MySQL binlog）、工具选择、存储路径、保留周期、压缩、cron。
- 执行模式：定时执行与手动立即执行。
- 执行前检查：磁盘空间、实例可达、工具可用。
- 日志记录：开始/结束时间、大小、结果、错误信息。
- 历史管理：筛选、下载、删除、恢复演练指引。

### 3.7 认证与权限

- 认证模式：本地用户 / LDAP（配置切换）。
- 操作审计：记录所有关键操作。
- API 统一鉴权：JWT。

## 4. 非功能需求

- 页面加载 <= 3 秒；监控刷新 <= 5 秒。
- 高并发与连接池管理，监控采集不得明显影响业务库。
- 模块化扩展，后续可新增 PostgreSQL 等数据库类型。
- 安全要求：
  - 连接密码使用 Fernet 加密存储。
  - LDAP 认证走 TLS。
  - 敏感操作带审计日志。
