# 系统架构设计

## 1. 分层

- `frontend`：Vue SFC + Element UI 风格界面（实现使用 Element Plus）。
- `backend`：Flask REST API，负责鉴权、元数据管理、监控采集、备份调度。
- `metadata-db`：MySQL，存储实例信息、策略、监控快照、审计日志。
- `agent-less collection`：后端通过驱动/命令直连目标数据库采集。

## 2. 核心模块

- `auth`：本地账户与LDAP登录，JWT签发。
- `asset`：数据库实例/集群增删改查。
- `dns`：域名解析刷新任务与手动刷新。
- `monitor`：采集器框架，按数据库类型收集指标。
- `backup`：策略配置、任务执行、日志归档。
- `rbac`：管理员/普通用户权限控制。
- `audit`：关键操作记录。

## 3. 调度任务

- 每小时：域名解析刷新。
- 每5分钟：关键实例监控采样。
- 按策略 cron：备份任务执行。

## 4. 可扩展点

- 新数据库类型只需增加：
  - 数据库驱动接入
  - 指标采集器
  - 备份执行器（可选）
  - 前端类型页面配置

## 5. 安全策略

- 元数据库连接信息加密（Fernet）。
- JWT鉴权 + 角色校验。
- LDAP认证建议开启 LDAPS/TLS。
- 审计日志不可被普通用户删除。
