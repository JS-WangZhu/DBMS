# 数据库实例物理机探测设计

## 目标

为所有数据库实例提供物理机地址的手动维护与 vCenter 自动发现能力。自动发现以实例 IP 为输入：虚机位于已启用 vSAN 的集群时，物理机地址写入固定值 `vSAN`；否则写入虚机当前所在 ESXi 节点的 Management VMkernel IP。

系统通过后台定时任务批量查询 vCenter，支持总开关、统一轮询周期、多个 vCenter、单个 vCenter 启停、连接测试和手动立即探测。所有 vCenter 操作必须是只读查询，不得调用任何资产变更接口。

## 范围与约束

- 支持 MySQL、MongoDB、Redis、PostgreSQL、Doris 等所有共用 `db_instances` 的实例类型。
- 实例级探测模式为 `manual` 或 `auto`。
- 现有已填写物理机地址的实例保持手动模式，避免升级后被自动覆盖。
- vCenter 配置的虚拟机 IP CIDR 在全局范围内不得重叠，一个实例 IP 只归属一个 vCenter。
- 失败时保留实例上一次成功的物理机地址，失败原因集中展示在物理机探测管理页面。
- vCenter 账号应授予只读角色；应用代码仅使用登录、PropertyCollector/容器视图查询和退出能力。
- 不提供虚机迁移、重配置、开关机、快照、Host 配置等操作，也不封装对应接口。

## 方案选择

采用“按 vCenter 分组批量拉取”。每轮先按 CIDR 将自动发现实例分组，每个 vCenter 建立一次会话，批量读取虚机网络、运行宿主、集群 vSAN 状态和宿主管理网络信息，在内存中匹配并统一回写。

未采用逐实例查询，因为它会产生重复登录和资产扫描；未采用完整本地资产镜像，因为当前需求不需要资产版本、快照过期和全量 CMDB 能力。

## 数据模型

### 全局探测配置

单行配置包含：

- `enabled`：显式总开关，默认关闭。
- `poll_interval_minutes`：统一轮询周期。
- `connect_timeout_seconds`：vCenter 连接/查询超时。
- `batch_size`：单批处理的最大实例数，用于控制内存和事务规模。

关闭总开关后移除定时任务，但保留 vCenter 配置与探测历史。立即探测仍要求总开关已开启，避免绕过显式停用意图。

### vCenter 配置

每条配置包含：名称、地址、端口、一个或多个 CIDR、用户名、加密密码、TLS 证书校验开关、启用状态、最近连接测试结果与时间。密码使用项目现有加密能力存储，列表和详情 API 只返回“已配置”标记，不返回密文或明文。

保存时规范化 CIDR，并校验全部启用或停用配置之间都不存在重叠，防止未来启用配置时产生路由歧义。地址与端口组合应唯一。

### 实例字段

在实例的 `extra_json` 中维护：

- `physical_discovery_mode`: `manual` 或 `auto`，缺省按 `manual` 处理。
- `physical_address`: 手动值或自动发现的最后成功值。
- `physical_discovery_source`: 最近成功匹配的 vCenter 标识，仅自动模式使用。
- `physical_discovered_at`: 最近成功时间，仅自动模式使用。

实例切换到自动模式后，页面禁止手工编辑物理机地址；切回手动模式时保留当前值并允许修改。

### 探测运行与明细

运行记录包含触发方式（定时/手动）、目标 vCenter、开始与结束时间、运行状态、实例总数、成功数、失败数、未匹配数和汇总错误。

明细记录包含运行 ID、实例 ID、实例名称快照、输入 IP、结果状态、发现结果、错误代码、脱敏错误信息和发生时间。历史记录支持按时间、vCenter、状态和实例筛选及分页。

## 后台组件

- 配置服务：CRUD、CIDR 规范化与重叠校验、密码加解密、调度同步。
- vCenter 只读客户端：定义最小查询接口；生产适配器使用 pyVmomi，测试使用假客户端。
- 路由器：根据规范化 IP 与 CIDR 将实例唯一分配给 vCenter。
- 探测编排器：防重入、分组、串行查询、匹配、记录和批量回写。
- APScheduler 任务：按全局周期注册，使用 `max_instances=1` 和 `coalesce=True`。
- REST API：配置管理、连接测试、立即探测、运行记录和失败明细。

## 探测数据流

1. 定时任务或手动操作进入同一编排器。进程内互斥与调度器防重入共同保证同一时间最多一个探测周期；冲突请求返回“探测进行中”。
2. 查询 `enabled=true` 且 `physical_discovery_mode=auto` 的实例。输入地址优先使用 `resolved_ip`，否则使用可解析为 IP 的 `host_input`；无法获得 IP 时直接记录失败。
3. 按全局无重叠 CIDR 将实例路由到唯一启用 vCenter。未命中网段时记录未匹配，不访问任何 vCenter。
4. vCenter 依配置顺序串行执行。每个配置建立一次会话，通过 PropertyCollector 或等效批量查询读取所需字段。
5. 使用 VMware Tools 上报的 `guest.ipAddress` 与 `guest.net[].ipAddress[]` 匹配实例 IP，定位虚机及其当前 `runtime.host`。
6. 判断 Host 所属 ComputeResource。若其集群 vSAN 配置明确为启用，则返回 `vSAN`；否则从 Host 的 Management 类型 VMkernel vNIC 取得 IP。若无法确定管理 IP，则失败，不以主机名代替 IP。
7. 每个 vCenter 完成匹配后，以短事务分批更新实例最后成功值和成功元数据，并写入运行/明细记录。单个 vCenter 失败不会中止其他配置。
8. 始终在 `finally` 中退出会话。任何失败均不清空旧物理机地址。

## vCenter 负载与只读安全

- 不逐实例登录或查询；每个 vCenter 每轮只建立一次会话并批量取数。
- vCenter 之间串行处理，不创建跨 vCenter 并发洪峰。
- 查询属性限定为匹配所需字段，不拉取无关配置与性能数据。
- 配置超时、批次上限、任务防重入和调度合并。
- 客户端接口不暴露任何变更方法；单元测试断言调用轨迹仅含连接、查询、退出。
- 连接测试只执行登录、最小只读查询与退出。
- 日志和 API 错误必须脱敏，不记录密码、会话 Token 或包含凭据的 URL。

## 前端设计

在“配置管理”下增加“物理机探测管理”及独立权限键。页面包含：

- 顶部全局开关、统一轮询周期、超时和批量大小。
- vCenter 配置表与新增/编辑弹窗；支持连接测试、单个启停和立即探测。
- 探测运行记录表；支持汇总状态筛选、失败详情查看和分页。

所有实例编辑页在物理机地址旁新增“手动填写/自动发现”选择。手动模式展示可编辑输入框；自动模式展示只读的最后成功结果、来源和成功时间。

## API 与权限

新增物理机探测管理权限键，导航、页面和全部管理 API 均受该权限控制。实例原有编辑权限控制实例探测模式的修改。

API 提供：

- 读取/更新全局配置。
- vCenter 列表、新增、更新、删除、启停和连接测试。
- 对单个 vCenter 立即探测。
- 探测运行列表与运行明细分页。

删除仍被历史记录引用的 vCenter 时采用软删除或保留名称快照，确保历史可读。立即探测采用同步接受、后台执行的方式返回任务记录 ID，避免 HTTP 请求长时间占用。

## 错误处理

错误按配置校验、无 CIDR 匹配、连接/认证、证书、查询超时、虚机未找到、Management IP 不可确定和内部错误分类。用户页面显示可操作的脱敏信息；详细堆栈仅进入服务端日志且不得含凭据。

运行状态分为成功、部分成功、失败；部分实例失败不回滚同批已成功实例。数据库写入失败则回滚该短事务并记录运行级错误。

## 测试策略

- 单元测试：CIDR 格式与重叠、IP 路由、输入 IP 选择、vSAN 判定、Management VMkernel IP 提取、错误脱敏。
- 服务测试：手动模式不覆盖、自动模式成功回写、失败保留旧值、多 vCenter 串行、每个 vCenter 单会话、任务防重入。
- 只读契约测试：假 SDK 记录调用，断言仅连接、批量查询和退出；生产适配器不暴露变更入口。
- API 测试：配置 CRUD、密码掩码、权限、启停、连接测试、立即探测、运行及明细分页。
- 调度测试：总开关与周期变更后正确重建/移除任务，任务不重叠且合并错过执行。
- 前端测试与构建：实例模式联动、自动字段只读、配置与记录页面关键交互，以及完整生产构建。
- 回归测试：现有实例 CRUD、实例列表、状态采集和其他调度任务保持通过。

## 验收标准

- 可在配置管理中显式开启物理机探测并设置统一周期。
- 可维护多个 CIDR 不重叠的 vCenter，并单独启停、连接测试和立即探测。
- 所有数据库实例可选择手动填写或自动发现。
- 自动发现准确按实例 IP 找到虚机；vSAN 返回 `vSAN`，非 vSAN 返回当前 ESXi Management IP。
- 探测失败不清空上次成功值，页面可查失败原因与时间。
- 每个 vCenter 每轮使用单一会话批量查询，任务不并发重入。
- 代码与测试证明 vCenter 集成仅执行只读查询，不存在变更行为。
